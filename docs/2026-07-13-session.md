## What Was Built

### `src/harness/context.py`
- `deliver(message, workspace)` — Scans user message for `@path` references, reads files from workspace, injects content inline with `--- filename --- ... --- end filename ---` markers.
- Regex-based: `@` followed by path chars (word chars, dots, slashes, hyphens).
- Trailing sentence punctuation stripped from captured path.
- Security: workspace confinement via `os.path.abspath()` boundary check.
- Graceful fallback: missing, empty, unreadable, or out-of-workspace paths leave `@ref` unchanged.

### `src/harness/agent.py`
- Added import: `from harness.context import deliver`
- In `send()`: `resolved = deliver(message, self.workspace)` before storing in `self.messages`

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Trailing punctuation | `rstrip(".,!?;:")` on captured path | Prevents sentence-ending period from being part of file path |
| Injection format | `--- filename ---\ncontent\n--- end filename ---` | Clear separation, easy for model to parse |
| Where injection happens | Before storing in `self.messages` | History stores resolved version — model sees file content on replay too |
