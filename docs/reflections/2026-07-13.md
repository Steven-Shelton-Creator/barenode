## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Trailing punctuation | `rstrip(".,!?;:")` on captured path | The first test caught this — `@b.txt.` was matching `b.txt.` instead of `b.txt`. Simple, correct fix. |
| Injection markers | `--- filename ---` wrapper | Clear visual boundary. Easier for the model to distinguish file content from the user's question. |
| Injection in stored message | Resolve before `self.messages.append()` | History stores the fully resolved version. On replay, the model sees the file content directly without re-reading. |

---

## Surprises / Learnings

- **The first test failure caught a real edge case.** `@a.txt with @b.txt.` — the trailing period after `@b.txt` was captured as part of the path. The regex `[\w.\-/]+` greedily includes `.`. The fix (`rstrip`) was simple, correct, and well-covered by the test.
- **The real implementation was ~40 lines** of actual logic (the rest is imports, docstrings, and comments). The chapter matches its promise — one primitive, small code surface.
- **Wiring into the existing architecture was frictionless.** The Agent's `send()` method was already designed to accept a `workspace` parameter and resolve messages. CH04 slotted in with 2 lines of new code.
- **The workspace concept from CH03 paid off immediately.** The workspace path was already threaded through the Agent class — `deliver()` just used it directly.
