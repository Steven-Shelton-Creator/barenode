# Phase 14 — Terminal UI

**Status:** 📝 Not Started

---

## Goal

Replace the bare REPL with a real terminal UI — two panes, live trace stream, approval modals. Complete the three-package model (model + harness + UI = full agent).

## Concepts

- **Two-pane TUI:** Conversation on the left, live trace stream on the right.
- **Approval modal:** Shows a unified diff for file edits, not just a bash command prompt.
- **Worker thread:** The UI runs the same agent in a worker thread and renders what the tracer records.
- **`textual` framework:** The UI package is the only one that imports textual.
- **Git work tree:** Works on a copy of the repository — the checkout stays untouched.

## Plan

1. Build `app.py` — textual app with two-pane layout.
2. Build `widgets.py` — conversation pane, trace stream pane, approval modal.
3. Run the agent in a worker thread, consume tracer events for live updates.
4. Approval modal shows diffs for file edits.

## Files
