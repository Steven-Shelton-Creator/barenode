**Status:** ✅ Complete
**Date:** 2026-07-15

## Goal

Replace the bare REPL with a real terminal UI — two panes, live trace stream, approval modals. Complete the three-package model (model + harness + UI = full agent).

## Concepts

- **Two-pane TUI:** Conversation on the left, live trace stream on the right.
- **Approval modal:** Shows a unified diff for file edits, not just a bash command prompt.
- **Worker thread:** The UI runs the same agent in a worker thread and renders what the tracer records.
- **`textual` framework:** The UI package is the only one that imports textual.
- **Git work tree:** Works on a copy of the repository — the checkout stays untouched.

## Plan

1. Build `app.py` — textual app with two-pane layout.
2. Build `widgets.py` — conversation pane, trace stream pane, approval modal.
3. Run the agent in a worker thread, consume tracer events for live updates.
4. Approval modal shows diffs for file edits.

## Learnings

### Textual Framework

Textual 8.x provides a clean async framework for terminal UIs. The `@work` decorator makes worker thread management trivial. The `ModalScreen` class provides a natural pattern for approval dialogs.

### Worker Thread Architecture

The agent runs in a `@work(exclusive=True, thread=True)` worker. The `call_from_thread()` method is used to update the UI from the worker thread. This prevents blocking the UI while the agent processes messages.

### Approval Modal

The approval modal shows a unified diff for file writes using Python's `difflib.unified_diff()`. When no existing content is available (new file), it shows the content as-is.

### Only UI Imports Textual

The `textual` dependency is only imported in `src/ui/`. The rest of the codebase (harness, model) has no knowledge of the UI framework. This proves the three-package architecture works.
