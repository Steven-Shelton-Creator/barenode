## What Happened

### The Surprise

The code was never missing — we were looking at the wrong git ref. A detached HEAD at `CH01` tag meant the working tree reflected the codebase frozen at Chapter 1. All CH02–CH04 commits existed on `master` but were invisible because the checked-out ref was the wrong one.

### The Fix

1. `git checkout master` — resolved instantly
2. `git branch --show-current` confirmed `master`
3. All CH02–CH04 code verified present

### Guardrails Added

Two layers of protection in `AGENTS.md`:
- **Identity section** — quick git discipline check visible on every session load
- **Orientation section** — detailed branch verification with rules and commands

## Key Takeaway

The session logs, schema map, and changelog all reference the correct state. The **only thing that was wrong** was the checked-out ref. This is a classic Git workflow issue: tags are for reading, branches are for building. The new guardrails make this explicit.

### The Surprise

**Circular import trap.** The dependency chain `tools.py → subagent.py → agent.py → tools.py` refused to load. Fixed with lazy imports — wrapper functions that do a local import inside the function body instead of at module level.

### Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Parallel execution | `ThreadPoolExecutor` | Simpler than asyncio, no event loop management |
| Session isolation | Thread-safe counter | Prevents file collisions during parallel fan-out |
| Error handling | Per-subtask try/except | One failure doesn't crash the batch |
| Approval gate | Both tools require approval | Spawning agents is a dangerous operation |

### The Verification Flow

```
Code file written → Arm gate → Inject "run tests" prompt → Agent calls bash
→ Check exit code → Pass (reset) or Fail (re-arm)
```

### Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Detection method | Check tool call args for `write_file` + code extension | No file I/O, avoids false positives |
| Result parsing | Parse `[exit code: N]` from bash output | Leverages existing sandbox format |
| State machine | Idle → Armed → Waiting | Prevents re-injection loops |
| Prompt injection | Append user message to `self.messages` | Agent sees it as natural instruction |
